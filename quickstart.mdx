Get the Barrett AI backend running, embed the chat widget, and send your first message — all in a few steps.

## Prerequisites

- Python 3.9 or later
- A [Cerebras API key](https://cloud.cerebras.ai) — Barrett AI uses `llama-3.3-70b` via `langchain-cerebras`
- The source files: `chatbot.py`, `chatbot.js`, the `app/` directory, and the `static/` directory

---

<Steps>

<Step title="Install dependencies">

From the project root, install all required packages:

```bash
pip install -r requirements.txt
```

`requirements.txt` includes: `fastapi`, `uvicorn`, `python-dotenv`, `langchain`, `langchain-cerebras`, `requests`, `httpx`, `pydantic`, `pydantic-settings`, `dataclasses-json`, `orjson`, `pyyaml`, `tenacity`, `packaging`, and `httptools`.

<Tip>Use a virtual environment to keep dependencies isolated: `python -m venv .venv && source .venv/bin/activate`</Tip>

</Step>

<Step title="Set your API key">

Barrett AI reads your Cerebras API key from the environment. Create a `.env` file in the project root:

```bash .env
CEREBRAS_API_KEY=your_cerebras_api_key_here
```

The key is loaded in `app/langchain_setup.py`:

```python app/langchain_setup.py
streaming_llm = ChatCerebras(
    api_key=os.getenv("CEREBRAS_API_KEY"),
    model="llama-3.3-70b",
    temperature=0,
    streaming=True,
    callbacks=[],
    verbose=False,
)
```

<Warning>Never commit `.env` to version control. Add it to `.gitignore` before your first commit.</Warning>

</Step>

<Step title="(Optional) Update CORS origins">

By default, `chatbot.py` allows requests only from these origins:

```python chatbot.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://172.16.40.27",
        "http://localhost",
        "http://172.16.0.50:8050",
        "http://172.16.0.51",
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "OPTIONS"],
    allow_headers=["*"],
)
```

Add any additional internal hostnames or IPs where you plan to embed the widget to this list before deploying.

<Note>If you are testing locally with `http://localhost`, it is already in the allowed origins list and no change is needed.</Note>

</Step>

<Step title="Start the server">

Launch the FastAPI app using Uvicorn. The `app` object lives in `chatbot.py`:

<CodeGroup>

```bash Default (development)
uvicorn chatbot:app
```

```bash Custom host and port
uvicorn chatbot:app --host 0.0.0.0 --port 81
```

```bash With auto-reload (development)
uvicorn chatbot:app --reload
```

</CodeGroup>

When the server starts successfully you will see output similar to:

```
INFO:     Application started with optimizations
INFO:     Uvicorn running on http://127.0.0.1:8000
```

Static files are served from the `/static` route, mapped to the `static/` directory.

</Step>

<Step title="Verify the server is responding">

Send a quick health check to the root endpoint:

```bash
curl http://localhost:8000/
```

Expected response:

```json
{
  "status": "ok",
  "ip": "127.0.0.1",
  "host": "localhost"
}
```

</Step>

<Step title="Configure the widget">

Open `chatbot.js` and update the two configuration variables at the top of the file to match your server:

```javascript chatbot.js
const ip_address = "10.10.10.10";
const backend_port = "81";
```

Set `ip_address` to the IP or hostname of the machine running the FastAPI server, and `backend_port` to the port Uvicorn is listening on. The widget builds the backend URL as:

```javascript
var s = `http://${ip_address}:${backend_port}`;
```

</Step>

<Step title="Embed the widget in your HTML page">

Add three tags to the `<head>` (or before `</body>`) of any internal page where you want the chatbot to appear:

```html
<!-- Font Awesome icons (required for widget buttons) -->
<link
  rel="stylesheet"
  href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0-beta3/css/all.min.css"
/>

<!-- Markdown rendering for AI responses -->
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>

<!-- Barrett AI widget -->
<script src="chatbot.js"></script>
```

No other markup is needed. The widget creates and injects all its own DOM elements when the page loads.

<Note>
`chatbot.js` must be served from the same host as the page embedding it, or from a path the browser can reach. If you are serving it from the FastAPI server, place it inside the `static/` directory and reference it as `http://<your-server>/static/chatbot.js`.
</Note>

</Step>

<Step title="Send your first message">

Open the page in a browser. You will see a red robot-icon button in the bottom-right corner. Click it to open the chat window.

Barrett AI will greet you with:

> **Hi! What is your name?**

Type your name and press **Enter** (or click the send button). The chatbot will address you by name for the rest of the session.

After providing your name, ask an IT question to confirm the LLM is responding:

> My Outlook is not sending emails. What should I do?

You should see Barrett AI stream a response token-by-token.

<Note>Sessions expire after 3 minutes of inactivity. The widget will disable the input and display "Session expired. Refresh page to start new chat." when this happens.</Note>

</Step>

</Steps>
