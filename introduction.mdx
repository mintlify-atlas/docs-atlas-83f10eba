## What Barrett AI does

Barrett AI handles first-line IT support for employees at Barrett Hodgson Pakistan. When a user opens the widget, Barrett AI greets them by name and helps them troubleshoot common IT problems — or escalates to the IT team when needed.

## How it works

Barrett AI is a FastAPI application that wraps a LangChain conversation chain powered by the Cerebras `llama-3.3-70b` model. Each user gets a session with windowed conversation memory, and responses stream token-by-token to the browser.

```
User Browser  ──→  chatbot.js widget
                        │
                        ▼
              FastAPI (chatbot.py)
                        │
              LangChain chain (langchain_setup.py)
                        │
              Cerebras LLM (llama-3.3-70b)
                        │
              StreamingResponse ──→  Browser
```

- **Sessions**: Each browser session gets a unique ID. Memory is held server-side for 3 minutes of inactivity, then purged.
- **Streaming**: Responses stream token-by-token via `StreamingResponse` for a natural typing effect.
- **History**: All conversations are saved as JSONL files on disk, organized by date, IP, and username.
- **Escalation**: If Barrett AI cannot resolve an issue, it directs users to call IT at Ext: 2025 or file a ticket at HDMS.

## Architecture overview

| Component | Technology |
|-----------|-----------|
| Backend framework | FastAPI + Uvicorn |
| LLM | Cerebras `llama-3.3-70b` via `langchain-cerebras` |
| Memory | LangChain `ConversationBufferWindowMemory` (k=6) |
| Caching | In-memory LLM cache + LRU hostname cache |
| Chat history | JSONL files written via async batched `FileOperationManager` |
| Frontend widget | Vanilla JavaScript (no dependencies except marked.js) |
| Load testing | Locust |

<Note>
Barrett AI is designed for internal network deployment. CORS is pre-configured for specific internal IP ranges. Update the allowed origins in `chatbot.py` before deploying to your environment.
</Note>
