## Python version

Python **3.10 or later** is recommended. Barrett AI uses:

- `match` statements (Python 3.10+)
- `asyncio.TaskGroup` patterns and improved asyncio event loop handling
- `asynccontextmanager` for lifespan management

<Warning>
  Python 3.9 and earlier are not supported. Install Python 3.10+ before proceeding.
</Warning>

## API key

Barrett AI uses **Cerebras** as its LLM provider. You need a Cerebras API key before the server will start.

<Steps>
  <Step title="Sign up for Cerebras">
    Create an account at [cerebras.ai](https://cerebras.ai) and navigate to the API section of your dashboard.
  </Step>
  <Step title="Generate an API key">
    Create a new API key and copy it. You will add this to your `.env` file in the configuration step.
  </Step>
</Steps>

<Info>
  The API key is read at startup via `os.getenv("CEREBRAS_API_KEY")`. The server will fail to initialize the LLM if this variable is missing or invalid.
</Info>

## Network requirements

The server binds to `0.0.0.0` and must be reachable from all client browsers on the configured port (default **81**).

CORS is enforced with an explicit allowlist. Out of the box the following origins are permitted:

```
http://172.16.40.27
http://localhost
http://172.16.0.50:8050
http://172.16.0.51
```

Any browser origin not in this list will have its requests blocked. See [Configuration](/deployment/configuration) for how to update the allowlist.

- **`static/`** — Must be present before startup. Contains `logo.png`, which is served at `/static/logo.png` and referenced by the chat widget.
- **`logs/`** — Created automatically on first run by the logging setup. Log files rotate at 5 MB with 3 backups; files older than 7 days are deleted.
- **`chats/`** — Created automatically on startup (`HIST_DIR.mkdir(exist_ok=True)`). Chat history is written here, organized by date.

<Tip>
  Create `logs/` and `static/` before first run to avoid permission errors:

  ```bash
  mkdir -p logs static
  ```
</Tip>

## Windows vs Linux

Barrett AI runs on both Windows and Linux. The Windows `ProactorEventLoop` is configured automatically:

```python
if hasattr(asyncio, "WindowsProactorEventLoopPolicy"):
    asyncio.set_event_loop_policy(asyncio.WindowsProactorEventLoopPolicy())
```

No manual configuration is needed for Windows. On Linux this block is a no-op.

<Note>
  Production deployments should prefer Linux for better asyncio performance and compatibility with process managers like `systemd` or `supervisor`.
</Note>
