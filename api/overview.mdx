The Barrett AI REST API is a FastAPI application that exposes endpoints for chat, session management, and IP tracking. All endpoints are served over HTTP on your internal network.

## Base URL

The server binds to the host and port you configure when starting Uvicorn:

```
http://{ip_address}:{port}
```

For example, if Barrett AI runs on `172.16.0.50` on port `81`:

```
http://172.16.0.50:81
```

No trailing slash is required. The `chatbot.js` widget builds this URL from the two variables at the top of the file:

```javascript chatbot.js
const ip_address = "10.10.10.10";
const backend_port = "81";
var s = `http://${ip_address}:${backend_port}`;
```

## Authentication

<Note>
Barrett AI has no API key or token-based authentication. It is designed exclusively for internal network deployment and relies on CORS policy and network-level access control to restrict access.
</Note>

Do not expose the API port on a public interface without adding an authentication layer.

## CORS

Cross-origin requests are restricted to a fixed list of allowed origins configured in `chatbot.py`:

```python chatbot.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://172.16.40.27",
        "http://localhost",
        "http://172.16.0.50:8050",
        "http://172.16.0.51",
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "OPTIONS"],
    allow_headers=["*"],
)
```

Add the origin of every page that will embed the widget to this list before deploying.

## Content type

All `POST` endpoints require the `Content-Type: application/json` header. The `/chat` endpoint explicitly validates this header and returns `400` if it is missing or incorrect.

## Client IP detection

Every request handler that needs the caller's IP address calls `get_client_ip()` from `app/utilities.py`. The function checks headers in order:

1. `X-Forwarded-For` — first address in the comma-separated list (set by reverse proxies)
2. `X-Real-IP` — set by Nginx and similar proxies
3. `request.client.host` — direct TCP connection address

```python app/utilities.py
def get_client_ip(request: Request) -> str:
    fwd = request.headers.get("x-forwarded-for")
    real = request.headers.get("x-real-ip")
    ip = fwd.split(",")[0].strip() if fwd else real or request.client.host
    return ip
```

## Response formats

The API uses two response formats depending on the endpoint and context:

| Format | Content-Type | Used by |
|--------|-------------|--------|
| JSON | `application/json` | All endpoints except normal chat turns; error responses |
| Streaming text | `text/plain` | `/chat` for normal assistant replies after username is set |

### JSON responses

Successful JSON responses follow this general shape:

```json
{
  "status": "success",
  "message": "Human-readable description"
}
```

Error responses use FastAPI's default `HTTPException` format:

```json
{
  "detail": "Error description"
}
```

### Streaming responses

Normal chat replies stream as `text/plain`. The response body sends tokens one by one as they are generated by the Cerebras LLM. Read the stream with a `ReadableStream` reader in the browser:

```javascript
const reader = response.body.getReader();
const decoder = new TextDecoder();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value, { stream: true });
  // append chunk to the chat UI
}
```

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Health check — returns server status, caller IP, and resolved hostname |
| `GET` | `/check-ip` | Check whether the caller's IP is in the stored IP set |
| `POST` | `/store-ip` | Add an IP address to the in-memory stored IP set |
| `POST` | `/reset-session` | Clear conversation memory and mark a session as expired |
| `POST` | `/chat` | Send a message and receive a streamed or JSON response |
