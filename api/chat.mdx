---
title: "POST /chat"
description: "Send a message to Barrett AI and receive a streaming or JSON response."
---

Send a message in an active session. Depending on the session state, the endpoint returns either a streaming `text/plain` response (for normal assistant replies) or a JSON response (for session events such as expiry and the first-message name exchange).

## Request

**Method:** `POST`  
**Path:** `/chat`  
**Content-Type:** `application/json` (required — the endpoint validates this header)

### Headers

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`. Returns `400` if missing or set to any other value.
</ParamField>

### Body

<ParamField body="session_id" type="string" required>
  Unique identifier for the user's session. Generated by the widget on page load and stored in the browser. Must be a non-empty string.
</ParamField>

<ParamField body="message" type="string" required>
  The message text sent by the user. Must be a non-empty string after stripping whitespace.
</ParamField>

## Conversation flow

Every new `session_id` starts a two-step onboarding sequence before normal chat begins:

<Steps>
  <Step title="First message (greeting)">
    The first message sent with a new `session_id` — regardless of content — triggers Barrett AI to ask for the user's name. The response is JSON, not a stream.
  </Step>
  <Step title="Name message">
    The second message is treated as the user's name. Barrett AI acknowledges it and sets the username for the session. The response is JSON.
  </Step>
  <Step title="Normal chat">
    All subsequent messages are answered by the Cerebras LLM with streaming token-by-token output.
  </Step>
</Steps>

## Response types

### Streaming response (normal chat)

Returned for all messages after the username has been set. The response body is `text/plain` and tokens arrive one by one as the LLM generates them.

```
Content-Type: text/plain
```

Read the stream in JavaScript using a `ReadableStream` reader:

```javascript chatbot.js
const response = await fetch(`${s}/chat`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ session_id: sessionId, message: userMessage }),
});

if (response.body) {
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    const chunk = decoder.decode(value, { stream: true });
    // append chunk to the message bubble in the UI
  }
}
```

### JSON — session expired

Returned when the `session_id` has been marked as expired (either by inactivity timeout or by a prior call to `/reset-session`).

<ResponseField name="response" type="string">
  Human-readable message: `"Session expired; please refresh."`
</ResponseField>

<ResponseField name="session_expired" type="boolean">
  Always `true` in this response. The widget uses this flag to disable the input and show an expiry notice.
</ResponseField>

```json
{
  "response": "Session expired; please refresh.",
  "session_expired": true
}
```

### JSON — first message (greeting)

Returned for the very first message in a new session. Barrett AI asks the user for their name.

<ResponseField name="response" type="string">
  The greeting prompt: `"Please tell me your name."`
</ResponseField>

<ResponseField name="is_first_message" type="boolean">
  Always `true` for the first two messages (greeting and name confirmation).
</ResponseField>

```json
{
  "response": "Please tell me your name.",
  "is_first_message": true
}
```

### JSON — name confirmation

Returned after the user provides their name (the second message in a session). Barrett AI acknowledges the name and the session is ready for normal IT questions.

<ResponseField name="response" type="string">
  Acknowledgement message addressing the user by the name they provided.
</ResponseField>

<ResponseField name="is_first_message" type="boolean">
  `true` — this flag is set for both the greeting and the name-confirmation response.
</ResponseField>

```json
{
  "response": "Nice to meet you, Alex! How can I help you with IT today?",
  "is_first_message": true
}
```

## Error responses

| Status | Detail | Cause |
|--------|--------|-------|
| `400` | `Content-Type must be application/json` | Request sent without the correct `Content-Type` header |
| `400` | `Missing session_id or message` | Either `session_id` or `message` is absent or blank in the request body |

## Examples

<CodeGroup>

```bash Normal chat (streaming)
curl -X POST http://172.16.0.50:81/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "session_abc123", "message": "My Outlook is not sending emails."}'
```

```bash First message (greeting)
curl -X POST http://172.16.0.50:81/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "session_new456", "message": "Hello"}'
```

```bash Name message
curl -X POST http://172.16.0.50:81/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "session_new456", "message": "Alex"}'
```

</CodeGroup>

<CodeGroup>

```javascript JavaScript — streaming fetch
async function sendMessage(sessionId, message) {
  const response = await fetch(`http://172.16.0.50:81/chat`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ session_id: sessionId, message }),
  });

  const contentType = response.headers.get("Content-Type") || "";

  if (contentType.includes("application/json")) {
    // Session events (expired, first message, name confirmation)
    const data = await response.json();
    if (data.session_expired) {
      // Disable input and show expiry notice
      return;
    }
    // Display data.response in the chat UI
    return;
  }

  // Normal streaming reply
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    const chunk = decoder.decode(value, { stream: true });
    // Append chunk to the current message bubble
  }
}
```

</CodeGroup>

<Warning>
After receiving `session_expired: true`, do not re-use the same `session_id`. Generate a new one and reload the widget to start a fresh session.
</Warning>
